# Excel 导入字段映射说明（基于真实样本）

更新时间：2026-04-10  
适用文件：`docs/一级.xlsx`、`docs/二级.xlsx`

## 1. 文档目的

本说明用于定义“真实 Excel -> 系统数据模型”的导入映射与清洗规则，确保：

- 首页三项摘要可由 `一级.xlsx` 稳定生成。
- 详情页联系人可由 `二级.xlsx` 稳定生成。
- Excel 列名与页面/接口解耦，后续模板变化只改导入层。

## 2. 与既有方案的一致性结论

整体技术方案不变（Next.js + Prisma + MySQL）。

- `一级.xlsx` 可支撑首页摘要（领导/中层/人员）。
- `二级.xlsx` 可支撑详情页（单位下收费站当日值班人/职务）。
- 需要在导入层新增：结构归一化、单位别名映射、异常值兜底。

## 3. 样本结构概览

### 3.1 一级.xlsx

- 工作表数量：33（通常“每单位一个 sheet”）。
- 标准列：
  - 第1列：值班时间
  - 第2列：值班领导及联系方式
  - 第3列：值班中层及联系方式
  - 第4列：值班人员及联系电话
- 异常：
  - `泰曲路运管中心`、`高新材料公司`、`山东通航公司` 仅模板行，无实际日期数据，且存在第5列“值班周期”。
  - `投资发展公司`、`山高发展公司`、`深圳投资公司`、`畅赢投资公司`、`云创保理公司（速链云公司）`、`毅康科技公司` 仅 9 天数据（周末/轮班样式），非整月 30 天。

### 3.2 二级.xlsx

- 工作表：`Sheet1`（单表）。
- 结构：月度宽表。
  - 第1行：标题（如“2026年4月份收费站值班表”）。
  - 第2行：表头（序号、运管中心、收费站、1日...30日）。
  - 第3行起：数据。
- 列特征：
  - 运管中心列存在合并单元格，需要前向填充。
  - 日值班列为“姓名/职务”为主，少量异常值如“无”“关停”“于涛 管理员”。
  - 不包含电话字段。

## 4. 系统目标模型（导入落库视角）

## 4.1 units（单位主数据）

建议字段：

- `id`
- `name`（标准单位名称）
- `slug`
- `is_active`

说明：单位标准名以系统字典为准，不直接依赖 Excel 原始写法。

## 4.2 duty_overviews（首页摘要）

建议字段：

- `id`
- `unit_id`
- `duty_date`
- `leader_name`
- `leader_phone`（可空）
- `middle_name`
- `middle_phone`（可空）
- `staff_name`
- `staff_phone`（可空）
- `source_batch_id`

## 4.3 duty_contacts（详情联系人）

建议字段：

- `id`
- `unit_id`
- `duty_date`
- `department_name`（二级场景下存收费站名称）
- `contact_name`
- `contact_phone`（可空）
- `role`（职务，可空）
- `source_batch_id`

## 4.4 import_batches（导入批次）

建议字段：

- `id`
- `source_file_name`
- `source_month`
- `status`
- `total_rows`
- `success_rows`
- `failed_rows`
- `error_report`
- `created_at`

## 5. 一级.xlsx 映射规则（首页摘要）

单位粒度：每个 sheet 为一个候选单位。  
日期粒度：sheet 中每一条日期行为一条 `duty_overviews`。

字段映射：

- `sheet名称` -> `units.name`（先走标准化/别名映射）
- `值班时间` -> `duty_overviews.duty_date`
- `值班领导及联系方式` -> `leader_name` + `leader_phone`
- `值班中层及联系方式` -> `middle_name` + `middle_phone`
- `值班人员及联系电话` -> `staff_name` + `staff_phone`

姓名电话拆分规则（建议）：

1. 优先匹配 11 位手机号：`1\d{10}`。  
2. 若无 11 位手机号，但存在连续数字（如 8 位固话），作为电话写入。  
3. 数字前文本去空格后作为姓名。  
4. 无数字时：整串作为姓名，电话置空。

行过滤规则：

- 跳过标题行（`值班时间`）。
- 跳过空日期行。
- 允许非整月数据（如 9 天轮班），不判错。

## 6. 二级.xlsx 映射规则（详情联系人）

数据预处理：

1. 读取 `Sheet1`，表头使用第2行。  
2. 从第3行开始作为数据区。  
3. `运管中心` 前向填充（处理合并单元格）。  
4. 将 `1日值班人员/职务` ~ `30日值班人员/职务` 宽转长（unpivot）。

宽转长后标准字段：

- `center_name_raw`（运管中心）
- `station_name_raw`（收费站）
- `day_no`（1~30）
- `duty_text_raw`（如“张三/站长”）

再映射到系统字段：

- `center_name_raw` -> `units.name`（先标准化/别名映射）
- `source_month + day_no` -> `duty_contacts.duty_date`
- `station_name_raw` -> `department_name`
- `duty_text_raw` -> `contact_name` + `role`
- `contact_phone` 固定为空（待后续模板提供）

`duty_text_raw` 拆分规则：

1. 包含 `/`：左侧为 `contact_name`，右侧为 `role`。  
2. 不含 `/`：
   - 若值为“无”“关停”或空白，按“无值班记录”处理（不入联系人）。
   - 其他文本作为 `contact_name`，`role` 置空。

## 7. 单位名称对齐与别名映射

现状：一级与二级交集 20 个单位，存在命名不一致。

已确认需要映射：

- `利津桥` -> `利津大桥公司`

建议新增独立映射表（或配置）：

- `unit_aliases(alias_name, unit_id, enabled)`

匹配流程：

1. 先精确匹配标准单位名。  
2. 再匹配别名表。  
3. 仍未匹配则记导入错误并进入批次报告。

## 8. 异常与容错策略

## 8.1 一级异常

- 仅模板无数据的 sheet（如 `泰曲路运管中心`）不入 `duty_overviews`，但可保留 `units`。
- 9 天轮班 sheet 允许部分日期缺失。

## 8.2 二级异常

- 行尾可能出现“仅中心名、无收费站、无日值班”的尾行，直接忽略。
- `duty_text_raw` 为“无/关停”时不生成联系人。

## 8.3 电话异常

- 非 11 位数字（如固话短号）允许入库，不做前端强校验。
- 前端拨号链接仅对可识别号码启用，其他按普通文本展示。

## 9. 导入执行建议（落地顺序）

1. 先实现“解析器层”（一级解析器、二级解析器）。  
2. 再实现“标准化层”（单位名标准化、别名匹配、日期构造）。  
3. 再实现“入库层”（按批次写入 + 错误报告）。  
4. 最后接上传页触发导入流程。

## 10. 本阶段待确认项

1. `利津桥` 是否最终统一为 `利津大桥公司`（建议确认）。
2. `无/关停` 是否在详情页显示“当日无值班”，还是直接不显示（当前建议不显示联系人）。

4. 二级后续若新增电话列，是否同步回填 `duty_contacts.contact_phone`。
